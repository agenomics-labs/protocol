## Architecture critiques (cycle 2)

1. **The Phase 3 design closed AUD-004 (laundering loop) by removing the loop's input — i.e., by making the entire suspension state unreachable for new agents.** The `cleared_count` cost ladder, `MAX_CLEARED = 3`, the closed-state-machine invariant rule "Suspended ⇒ slash_count >= 3", and the `AgentSlashed` event all defend a state machine that no instruction can drive. Cycle 1 closed AUD-004 with an elegant cumulative-cost construct; cycle 2 reveals that the cost ladder is now a Maginot Line because PR-G silently severed every approach to it. **Either restore the slash path or formally retire the suspension model.** Do not ship to mainnet with this duality.
2. **The cross-program coupling between Settlement and Registry has grown more accounts (provider_profile, provider_owner_nonce, provider_authority, settlement_authority, protocol_config), but the validation discipline is asymmetric.** Settlement constrains `provider_authority = escrow.provider` strictly, then ships the other two as `UncheckedAccount` and trusts the Registry's seeds to do the work. The Registry does its job. But "the callee will validate" is a fragile invariant — any future Registry refactor that softens its seeds constraints (or any new Settlement caller that omits `provider_authority`) is one PR away from re-opening AUD-001. **Mirror the seeds constraints at the Settlement boundary as defense-in-depth.** See AUD-117.
3. **`verify_protocol_invariants` reaches across program boundaries by reading raw bytes.** This is a deliberate choice (avoiding a `settlement` crate dependency in the Registry), but it bakes Settlement's `ProtocolConfig` field order into the Registry's compiled bytecode silently. There's no compile-time check that the bytes-at-offset-8-are-32-bytes-of-pubkey-named-authority. This works today; it will fail silently on a future Settlement refactor. See AUD-104.
4. **The migration choreography in DESIGN-DECISIONS § "Ship sequence" cannot run because `migrate_agent_profile` cannot be successfully invoked (AUD-101).** The instruction's seeds derivation is wrong; every legacy profile is therefore unmigrated and unmigratable. The protocol's normalization story for AUD-001/002/004/007 hinges on this instruction working. **This should block the next mainnet deploy until fixed.**
5. **`reason` plumbing through CPI is a mess.** Settlement encodes "task_completed vs not" as a bool, the cpi.rs collapses two distinct reasons (dispute_loss, expiry_undelivered) into one code, and Registry exposes the conflated value to indexers. The whole reason-code design (0/1/2 + 3-255 reserved) is in a comment, not enforced. See AUD-108, AUD-109.
6. **Schema-bump-via-padding is clever (AUD-007 → `_reserved_aud007: [u8; 17]`) but the IDL exposes the padding as a public field.** Future SDK consumers iterating fields generically will see padding bytes. The padding is also writeable (Anchor doesn't gate `pub` fields by reserved-name convention). See AUD-110.
7. **`accept_task` boundary `now <= deadline` is a Phase-3 fix that closed AUD-009 but kept the equality edge case.** The grief vector is reduced but not eliminated. With the smaller post-PR-G slashes (-3 for expiry), the economic deterrent is weaker. See AUD-105 + AUD-107.

## Verdict

**The protocol is meaningfully safer at the unit-instruction level after cycle 1's Phase 3 PRs landed**: front-running governance is closed, the reputation policy is unified, the legacy unbounded `update_reputation` is gone, several grief vectors are tightened, the schema-migration headroom is in place, and several account-level constraints were hoisted into Anchor `#[account(constraint=...)]` blocks for defense-in-depth.

**But cycle 1 traded one class of issue for another at the architecture level:**
1. The slash path was excised wholesale; the suspension state machine is dead code (AUD-100). The entire AUD-004 closure now defends a state nothing can produce. Either restore slashing or formally retire it.
2. `migrate_agent_profile` is unreachable due to a seeds bug (AUD-101). The migration choreography that makes Phase 3's normalization story work cannot run. This is a hard mainnet blocker.
3. Settlement / Registry bound systems are 5 orders of magnitude apart; misconfiguration silently bricks slashing CPIs (AUD-102).
4. Cross-program raw-bytes reads couple programs invisibly (AUD-104, AUD-117). One Settlement field-order change away from silent breakage.
5. Two of the four design-mandated `assert_valid_profile` call sites are missing (AUD-103). Today inert; load-bearing the moment slashing returns.

The Phase 3 PRs were strong on closing the specific holes they targeted. They were weak on system-level invariants — the cleanup of the legacy reputation path took the slash → suspend mechanic with it, and the migration instruction's seeds were not regression-tested against the seed shape that every other context in the same file uses.

**Recommendation: do not promote to mainnet until AUD-100, AUD-101, and AUD-102 are closed. AUD-103, AUD-104, and AUD-117 should land in the same release window.** Track AUD-105 onward as cycle-2 follow-up.
