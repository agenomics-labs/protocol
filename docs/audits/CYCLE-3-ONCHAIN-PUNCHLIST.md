[^aud205]: Calibration finding, not a bug — closed by accepting ADR-131 (`docs/adr/ADR-131-sybil-cost-calibration.md`, Status: Accepted, this commit) as the auditable record that the launch parameters (`MAX_DELTA_PER_CALL = 10`, `SUSPEND_AT_SLASH_COUNT = 3`, no minimum-stake-at-registration, ADR-028 + ADR-097 economic / PDA-uniqueness defenses) were considered against AUD-205's `escrow > 3R + 3L` inequality and explicitly held at their current values for the launch window. **Threat-model boundary**: the calibration defends single-pair low-value sybil attempts at protocol-floor escrow values (`MIN_ESCROW_AMOUNT = 0.01 USDC`) and reputation-laundering via close-then-reopen on the same authority (ADR-097 nonce makes the post-close PDA address non-reusable); it does NOT defend capitalized sybil attackers transacting at high per-task escrow values, where `R ≈ 0.011 SOL` per-identity-rent dominates the right-hand side and any `E` two orders of magnitude above `R` flips the inequality into the attacker's favor. **Levers enumerated for post-launch (ADR-131 §"Levers available")**: lower `MAX_DELTA_PER_CALL`, raise `SUSPEND_AT_SLASH_COUNT` (with paired `assert_valid_profile` invariant update + migration cost), add minimum-stake-at-registration bond, tie escrow size to required reputation/stake, make the constants governance-tunable via `ProtocolConfig`. Each lever's cost is priced in the ADR; none are pulled today. **Re-calibration trigger** (ADR-131 §"Re-calibration trigger"): re-open this ADR when ANY of (a) observed sybil-pattern incidents on mainnet exceed 5 per quarter year-1 / 10 per quarter year-2+, (b) median escrow value sustains above 1 SOL on a 30-day rolling average, (c) a funded sybil incident is forensically reconstructed where `E > 3R + 3L` was reachable, (d) a new dispute-resolution surface lands that emits `reason in {1, 2}` from a code path other than `resolve_dispute` / `expire_escrow`. Closure mechanism is the Accepted ADR itself, not a code change. No IDL surface changes; no test-count change (this is a doc-only commit).
